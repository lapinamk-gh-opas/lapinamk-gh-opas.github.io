Projektitaulun luominen

Tässä harjoituksessa harjoitellaan GitHub Projects -työkalun käyttöä tehtävien hallinnan tukena. Tarkoituksena on nähdä, miten issueita seurataan projektitaululla sekä miten ne linkittyvät käytännön Git-työvaiheisiin.

Harjoitus 8 - GitHub Projects repositoriokohtaisen projektitaulun luominen
  1. Luo GitHub-repositorio tai hyödynnä olemassa olevaa
  2. Tarkista repositorion asetuksista, että Projects ominaisuus on käytössä (Settings → General → Features)
  3. Siirry repositorion Projects -välilehdelle
  4. Valitse Team Planning -template
  5. Nimeä taulu Harjoitus Team Planning ja älä importtaa vanhoja issueita
  6. Varmista, että Board -näkymässä on sarakkeet To do, In progress, Done
  7. Luo kolme issueta: Tee haara, Tee muutokset ja Tee PR
  8. Lisää issueille prioriteetti, koko, iteraatio ja vastuuhenkilö
  9. Toteuta tehtävät käytännössä
    1. Tee haara - Tee uusi haara repositorioon
    2. Tee muutokset - Lisää haaraan uusi tiedosto tai muokkaa olemassa olevaa tiedostoa
    3. Commitoi ja puske muutokset GitHubiin
    4. Tee PR - Tee Pull Request ja lisää sen kuvaukseen linkitys issueeseen: Tee PR (esim. Closes #3)
  10. Tarkista että Tee PR -issuen status päivittyi automaattisesti tilaan Done
  11. Lopuksi järjestä Backlog -näkymän sarakkeet ja luodut issuet, sekä niiden tiedot niin, että se vastaa alla olevaa kuvaa

Jos sait harjoituksen tehtyä onnistuneesti, hyvä sinä! Osaat käyttää GitHub Projects -työkalun perustoimintoja ja hallita tehtäviä visuaalisesti projektitaululla.

Vinkki! Voit hyödyntää GitHub Projects -taulua myös henkilökohtaisten tehtäviesi hallintaan tai pienissä tiimeissä työskentelyyn. Kokeile erilaisia näkymiä ja mukauta taulua tarpeidesi mukaan!
