GitHub Projects

GitHub Projects on GitHubin sisäänrakennettu työkalu projektinhallintaan. Sen avulla voit suunnitella, seurata ja organisoida kehitystyötä suoraan samassa ympäristössä, jossa lähdekoodi sijaitsee. GitHub Projects toimii kuten muutkin digitaaliset Kanban-taulut, mutta tarjoaa lisäksi integraatioita, automaatiota ja suoran yhteyden committeihin, brancheihin ja pull requesteihin.

Projektin luominen

GitHub Projectsin voi luoda joko omalle käyttäjätilille, repositorioon tai organisaatio kohtaisesti.

  1. Avaa GitHub-repositorio
  2. Siirry yläpalkin Projects-välilehdelle
  3. Valitse New Project ja nimeä projekti esimerkiksi “Git-harjoitukset”

Tehtävien lisääminen ja seuranta

Projektiin voidaan lisätä tehtäviä kahdella tavalla

  • Issueina – luodaan uusi tehtävä tai liitetään olemassa oleva issue tauluun
  • Draft itemeina – lisätään nopeita tehtäviä suoraan taululle ilman issuea

Jokaiselle tehtävälle voidaan määrittää vastuuhenkilö, prioriteetti ja deadline. Kun tehtävä etenee, se siirretään seuraavaan sarakkeeseen. GitHub päivittää taulua automaattisesti, jos tehtävä liitetään pull requestiin tai jos sen tila muuttuu issue-näkymässä

GitHub Projectsissa voi hyödyntää automaatiota ja näkymien suodatusta tehokkaaseen seurantaan

  • Views Luo erilaisia näkymiä (esim. taulukko, lista tai aikajana) projektin eri tarpeisiin
  • Filters Suodata tehtäviä vastuuhenkilön, tilan tai sprintin mukaan
  • Automation Määritä säännöt, kuten “Siirrä Done-sarakkeeseen, kun issue suljetaan”
