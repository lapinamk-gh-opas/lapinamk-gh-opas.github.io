GitHub Actions

GitHub Actions on GitHubiin sisäänrakennettu CI/CD-alusta, jonka avulla voidaan automatisoida ohjelmiston käännös-, testaus- ja käyttöönottoprosesseja. GitHub Actions ei suorita kaikkia tehtäviä itse, vaan ajaa määriteltyjä työnkulkuja (workflows) virtuaalikoneella (runner). Työnkulut määritellään YAML-tiedostoissa, joissa kuvataan, mitä työkaluja ja komentoja automaation tulee suorittaa.

Työnkulku käynnistyy aina, kun ennalta määritelty tapahtuma (event) toteutuu repositoriossa, kuten koodin push, pull request tai uuden issuen luonti. GitHub Actionsin avulla voidaan automatisoida monia toistuvia kehitys- ja ylläpitotehtäviä, myös sellaisia, jotka eivät suoraan liity CI/CD-putkeen, kuten esimerkiksi dokumentaation automaattiset päivitykset.

Miksi GitHub Actions

GitHub Actions tarjoaa helpon tavan automatisoida toistuvat työvaiheet ilman erillisiä CI/CD-työkaluja. Se vähentää manuaalista työtä, nopeuttaa kehitysprosessia ja parantaa työn toistettavuutta. Työnkulut voidaan rakentaa hyvin kevyistä yksittäisistä testiajoista aina monivaiheisiin julkaisuprosesseihin, joten ratkaisu skaalautuu sekä pieniin opiskelijaprojekteihin että laajoihin yrityssovelluksiin.

GitHub Actions - Keskeiset käsitteet ja komponentit

GitHub Actions -työnkulku muodostuu neljästä peruselementistä: tapahtumasta (event), tehtävistä (jobs), suoritusympäristöstä (runner) ja askeleista (steps). Näitä hyödyntämällä määritellään, milloin työnkulku käynnistyy, mitä siinä tehdään, missä ympäristössä se ajetaan ja mitä komentoja suoritetaan.

Työnkulut (Workflows)

GitHub Actions -työnkulku määritellään YAML-tiedostossa, joka sijaitsee repositorion juurihakemistossa polussa .github/workflows/. Jokainen työnkulku tallennetaan omaan tiedostoonsa, esimerkiksi .github/workflows/deploy.yaml.

Workflow toimii työnkulun pääohjeena: se määrittelee, milloin automaatio käynnistyy ja mitkä tehtävät siinä suoritetaan. Käynnistys voi tapahtua tapahtumasta, manuaalisesti tai ajastetusti, projektin tarpeiden mukaan.

YAML (YAML Ain't Markup Language) on rakenteinen tiedostomuoto, jossa sisennys kertoo tiedon hierarkian (kuten koodissa lohkot ilmaistaan sisennyksillä). GitHub Actionsin workflow-tiedostot (.yml tai .yaml) perustuvat YAML-syntaksiin, joten sisennyksen merkitys niissä on erittäin tärkeä. Sisennystaso kertoo, mihin avaimeen rivi kuuluu. Jos rivi on väärässä kohdassa, työnkulku ei välttämättä toimi lainkaan.

Perusperiaatteet:
  • YAML käyttää rakenteen määrittelyyn sisennystä
  • Sisennys tehdään aina välilyönneillä, ei sarkaimella (Tab)
  • Yleinen käytäntö GitHub Actions -tiedostoissa on käyttää kahta välilyöntiä per sisennystaso
  • Lisätietoa YAML-syntaksista löydät täältä.
Tapahtumat (Events)

Event määrittelee, milloin työnkulku käynnistyy. Se voi olla koodimuutos, pull requestin avaus, issuen luonti tai ajastettu suoritus (push, pull_request, issues, schedule). Event määritellään työnkulun YAML-tiedostossa, ja sen perusteella GitHub käynnistää workflow'n automaattisesti silloin, kun määritelty tapahtuma toteutuu.

Tehtävät (Jobs)

Tehtävä (job) on työnkulun itsenäinen kokonaisuus, joka suoritetaan omassa runner-virtuaaliympäristössään. Runneri määritellään tehtävän alussa, esimerkiksi ubuntu-latest. Job määrittelee, mitä vaiheita (steps) sen tulee suorittaa yhden ajon aikana, ja kaikki stepit ajetaan samassa runnerissa.

Suoritusympäristö (runner)
  • Runner on virtuaalikone, joka suorittaa työnkulun tehtävät
  • GitHub tarjoaa valmiit Linux, Windows ja macOS -virtuaaliympäristöt
  • Jokainen työnkulku ajetaan uudessa ja alustetussa (puhtaassa) virtuaalikoneessa
Askeleet (Steps)

Askeleet (steps) ovat jobin sisällä suoritettavia komentoja tai toimintoja. Ne ajetaan siinä järjestyksessä kuin ne on määritelty, ja ne voivat olla joko valmiita actioneita (kuten actions/checkout@v4) tai komentorivikomentoja.

Steppejä voidaan määrittää myös suoraan komentoina. Tätä käytetään esimerkiksi riippuvuuksien asentamiseen, testien suorittamiseen tai muihin tehtäviin.

Stepit muodostavat jobin varsinaisen toiminnallisuuden: ne määrittelevät kaikki toiminnot, jotka workflow'n aikana suoritetaan.

Toiminnot (Actions)

Action on uudelleenkäytettävä tehtävä, joka suorittaa tietyn toiminnon työnkulun aikana. Se voi olla valmis toiminto GitHub Marketplace -kirjastosta tai kehittäjän itse tekemä ratkaisu.

  • Esimerkkejä:
    • Koodin checkout GitHubista (actions/checkout), hakee lähdekoodi runnerin käyttöön
    • Julkaisun (release) tekeminen GitHubiin (actions/create-release@v1)
GitHub Actions -työnkulku rakentuu neljästä peruselementistä: event, jobs, runner ja steps.

Esimerkki työnkulun rakenteesta
on: 
  push                                # Event - mikä käynnistää workflow'n

jobs:                                 # Jobs - mitä workflow tekee
  build:                              # - tässä vain yksi job 'build'
    runs-on: ubuntu-latest            # Runner - missä jobi ajetaan

    steps:                            # Steps - mitä komentoja suoritetaan
      - uses: actions/checkout@v4     # Step 1 - ensin ajetaan valmis action
      - run: npm install              # Step 2 - sitten komentorivikomento

Tämä työnkulku käynnistyy aina push-tapahtumasta. Se ajaa build-nimisen tehtävän ubuntu-latest -ympäristössä, hakee projektin koodin käyttöön ja asentaa riippuvuudet komennolla npm install.
