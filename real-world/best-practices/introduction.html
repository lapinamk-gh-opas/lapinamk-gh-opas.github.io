Git ja GitHub - parhaat käytännöt

Tässä osiossa kootaan yhteen Gitin ja GitHubin parhaita käytäntöjä, eli best practices. Ne toimivat muistilistana tiimityön sujuvoittamiseen ja koodin hallinnan parantamiseen. Hyvien käytäntöjen noudattaminen auttaa pitämään repositorion selkeänä, yhteistyön tehokkaana ja projektin laadukkaana.

Repositorio

  • Pidä repositorio siistinä ja järjestyksessä
  • Lisää README-tiedosto, joka kuvaa projektin tarkoituksen
  • Käytä .gitignore-tiedostoa tiedostojen sulkemiseen pois versionhallinnasta
  • Huomioi yksityisyysasetukset ja varmista, että projektiin osallistuvilla henkilöillä on tarvittavat käyttöoikeudet

Haarat

  • Pidä päähaara aina käyttökelpoisena ja tuotantovalmiina
  • Käytä kehityshaaroja erillisiin ominaisuuksiin, bugikorjauksiin tai kokeiluihin
  • Nimeä kehityshaarat selkeästi ja kuvaavasti
  • Poista valmiit ja yhdistetyt haarat, jotta repositorio pysyy siistinä

Commitit

  • Commitoi pienissä, loogisissa osissa, jotta muutokset on helppo ymmärtää ja tarvittaessa perua
  • Kirjoita selkeät ja informatiiviset commit-viestit, jotka kuvaavat muutoksia

Push ja Pull

  • Vedä uusimmat muutokset ennen työn aloittamista ja ennen muutosten puskemista
  • Pidä kehityshaara ajan tasalla päähaaran kanssa, jotta vältyt konflikteilta
  • Puske muutokset säännöllisesti, jotta työ tallentuu etärepositorioon ja on muiden saatavilla

Tiimityö

  • Sovi tiimin kanssa yhtenäinen branch-strategia ja nimeämiskäytännöt
  • Käytä pull requesteja ja koodikatselmointeja ennen muutosten yhdistämistä
  • Hyödynnä projektinhallintaa (esim. GitHub Projects) työn organisointiin ja seurantaan

Turvallisuus

  • Käytä vahvoja salasanoja ja kaksivaiheista tunnistautumista GitHub-tililläsi
  • Älä tallenna salasanoja tai arkaluontoisia tietoja versionhallintaan
  • Hyödynnä GitHubin tarjoamia työkaluja, kuten Dependabottia, haavoittuvuuksien hallintaan

Nämä muodostavat hyvän peruspohjan mille tahansa projektille. Tiimit voivat lisäksi sopia tarkemmista käytännöistä projektin tarpeiden mukaan.
