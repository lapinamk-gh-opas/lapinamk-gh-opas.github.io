Miten haarat toimivat Gitissä

    • Siirry kansioon johon olet luonut git paikallisen repositorion komennolla:
      cd polku/kansioon/jossa/git-projekti/on
    • Git Bash osaa näyttää myös haaran jossa parhaillaan työskennellään.
      Kuten esimerkissä main
    • Luo uusi haara komennolla:
      git checkout -b haaran-nimi
    • Varmista, että haaran nimi vaihtui mainista antamaasi haaran nimeen.
    • Lisää uusi tiedosto komennolla:
      echo "Teksti sisältö tähän" >> tiedoston-nimi-tähän.md
    • Muokkaa olemassa olevaa README.md tiedostoa komennolla:
      echo "Lisättävä muutos tähän" >> README.md
    • Lisätään tiedostot stagin-tilaan komennolla:
      git add .
      • Tämän voisi tehdä myös komennolla:
        git add uuden-tiedoston-nimi.md muutetun-tiedoston-nimi.md
      • Kuitenkin mikäli muutokset koskevat useita tiedostoja, niin on helpompi lisätä kaikki tiedostot kerralla käyttäen komentoa: git add .
      • Käytä komentoa git add . varoen. Jos olet lisännyt samaan kansioon arkaluontoistatietoa sisältäviä tiedostoja, joita et halua lisätä repositioon, niin älä käytä git add . komentoa tai lisää repositorioon .gitignore -tiedosto, joka estää siihen nimettyjen tiedostojen lisäämisen repositorioon.
      • .gitignore:sta lisää tietoa osiossa: 2.5 yksityisyys ja tietoturva
    • Tehdään commit ja lisätään viesti komennolla:
      git commit -m "commit viesti tähän"
    • Viedään muutokset etärepositorioon ja luodaan sinne uusi haara:
      git push -u origin haaran-nimi

      • push vie muutokset etärepositorioon.
      -u asettaa paikallisen haaran seuraamaan etähaaraa.
      origin etärepositorion nimi (oletus nimi origin), viite repon url-osoitteesen.
      haaran-nimi eli haara jossa olet.

    • Mene selaimella GitHubiin
    • Klikkaa haaran valintaa jossa lukee Main
    • Valitse uusi haara
    • Aukeaa näkymä luodun haaran sisällöstä.
  • Haarojen yhdistämistä tässä esitellyllä tavalla voi käyttää pienissä projekteissa ja kehitystyötä yksin tehdessä, mutta tiiminä työskennellessä ja isompien projektien parissa kannattaa soveltaa osion 2.3 Pull Request oppeja.
    • Siirry main-haaraan komenolla:
      git checkout main tai git switch main
    • Varmista, että haaran nimi vaihtui taikaisin mainiksi.
    • yhdistä etärepositorion main-haaran viimeisin tila paikalliseen main-haaraan komennolla:
      git pull origin main

      Jos käytit aiemmin komentoa: git push -u origin main niin nyt riittää komento: git pull

    • Tuo paikallisen feature-haaran muutokset paikilliseen main haaraan komennolla:
      git merge yhdistettävän-haaran-nimi

      • Jos konflikteja ilmenee, Git ilmoittaa ja pyytää ratkaisemaan ne tässä kohtaa. Tästä lisää osiossa 2.4 merge-konfliktit

    • Viedään muutokset etärepositorioon komennolla:
      git push origin main

      • Tässäkin riittää komento: git push, mikäli -u lippua on käytetty.

    • Siirry selaimella GitHubiin ja klikkaa haaran valintaan käytettävää alasvetovalikkoa.
    • Valitse listasta main haara
    • Aukeaa näkymä main haaran sisällöstä, joka vastaa yhdistetyn haaran haaran sisältöä.
