Haarat

Haarat (tuttavallisemmin branchit) ovat Gitin keskeinen ominaisuus, joka mahdollistaa samanaikaisen kehitystyön ilman, että päähaara (main) häiriintyy. Päähaarasta voidana forkata eli haarauttaa erillinen kehityshaara (feature branch). Kehityshaarassa voidaan tehdä muutoksia, kokeilla uusia ominaisuuksia tai korjata bugeja ilman, että ne vaikuttavat suoraan päähaaraan.

Workflow kehityshaaran hallintaan

  1. Luo erillinen kehityshaara mainista ja nimeä se kuvaavasti (esim. login), jotta muutokset pysyvät erillään päähaarasta git checkout -b login
  2. Tee muutokset ja commitoi ne pienissä paloissa selkeillä viesteillä. Muista myös testata, että muutos toimii tuotantoa vastaavassa ympäristössä git push origin login
  3. Avaa pull request: kuvaa mitä teit ja miksi git request-pull origin/main!!!!TARKISTA ETTÄ TOIMII
  4. Keskustele tiimin kanssa ja tee tarvittavat muutokset
  5. Hanki hyväksyntä ja yhdistä main-haaraan git merge login
  6. Poista haara, jotta ympäristö pysyy hallittuna git branch -d login

Tarkemmat selitykset koodeista voit katsoa kappaleesta Perusteet / 1.3 Keskeiset komennot

Haarojen edut tiimityössä

Jokainen tiimin jäsen voi luoda oman haaransa tarpeellisille ominaisuuksille ja työskennellä siinä itsenäisesti. Kun työ on valmis ja testattu, muutokset voidaan yhdistää päähaaraan pull requestin avulla.

Branchien ansiosta tiimi voi kehittää useita ominaisuuksia samanaikaisesti, mikä lisää tuottavuutta ja vähentää konflikteja. Ne mahdollistavat myös koodin tarkastamisen ja hyväksymisen ennen muutosten käyttöönottoa, mikä parantaa laatua ja ehkäisee virheiden päätymistä tuotantoon.

Haarojen avulla tiimit voivat myös kokeilla uusia ideoita ja villejäkin ideoita ilman riskiä, koska erillinen päähaara pysyy turvassa ja käyttökelpoisena.
