Tiimityöskentelyn perusteet

Tiimityöskentely onnistuu parhaiten, kun tiimi on sopinut yhteiset pelisäännöt Gitin ja GitHubin käytölle. Pelisäännöt helpottavat yhteistyötä, ehkäisevät väärinkäsityksiä ja pitävät projektin hallittavana.

Tiimi voi esimerkiksi sopia seuraavista asioista
  • Miten projektin repositorio jaetaan ja kuka vastaa mistäkin osa-alueesta / toiminnosta
  • Millaisia commit-viestejä käytetään
  • Miten repositorioon tehdyt muutokset tarkastetaan ja hyväksytään
  • Miten ja missä tiimi kommunikoi työn etenemisestä

Kun yhteiset pelisäännöt on selvillä, tiimin on helpompi hyödyntää Gitin ja GitHubin tarjoamia työkaluja tehokkaasti.

Commitit

Edustavat yksittäisiä muutoksia koodiin ja sisältävät kuvauksen tehdyistä muutoksista.

Haarat

Mahdollistavat eri ominaisuuksien tai korjausten kehittämisen rinnakkain ilman, että ne häiritsevät päähaaraa.

Pull Requestit

Tarjoavat tavan tarkastella koodimuutoksia, keskustella niistä ja hyväksyä ne ennen yhdistämistä päähaaraan.

GitHub Projects

Projektinhallintatyökalu, joka auttaa tiimejä suunnittelemaan, seuraamaan ja hallitsemaan tehtäviä visuaalisesti.

GitHub Issues

Issues on GitHubin tikettijärjestelmä, jota käytetään tehtävien, bugiraporttien ja kehitysehdotusten kirjaamiseen.

GitHub Actions

Automaatiotyökalu, jolla voidaan toteuttaa CI/CD-putkia ja muita työnkulkuja suoraan GitHubissa.

GitHub Wiki

Keskitetty alusta projektin dokumentaatiolle, kuten käyttöohjeille ja kehitysprosesseille.

Security Features

Työkalut, jotka auttavat tiimejä hallitsemaan tietoturvaa, kuten riippuvuuksien tarkastukset ja haavoittuvuuksien skannaus.

Tutustumme näihin ja moniin muihin ominaisuuksiin ja työkaluihin tarkemmin oppaan edetessä.
