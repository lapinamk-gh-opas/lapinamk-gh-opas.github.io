Selkeät commit-viestit

Hyvät commit-viestit ovat tärkeitä, jotta tiimin jäsenet ymmärtävät tehdyt muutokset. Hyvä commit-viesti auttaa myös, kun halutaan tarkastella projektin historiaa tai etsiä tiettyjä muutoksia.

Commit on Gitin perusyksikkö, joka paketoi koodiin tehdyt muutokset ja niihin liittyvän commit-viestin yhteen. Näin syntyy selkeä tallenne, joka kertoo mitä on tehty ja milloin. Useista committeista muodostuu projektin historia, jota voidaan tarkastella ja palata tarvittaessa aiempiin versioihin.

Tässä muutamia vinkkejä hyvän commit-viestin kirjoittamiseen:

  • Käytä selkeää ja kuvaavaa viestiä, joka kertoo mitä muutoksia on tehty ja miksi
  • Käytä englanninkieltä (oletuskieli ohjelmistokehityksessä), ellei muuta erikseen sovita
  • Kirjoita viesti imperatiivimuodossa (esim. "Add new feature")
  • Jos olet tehnyt useita muutoksia, harkitse niiden jakamista useampaan commit-viestiin (esim. UI-korjaukset omaansa ja uuden toiminnon lisääminen omaansa)
  • Ensimmäisen rivin tulisi olla lyhyt yhteenveto (max. 50 merkkiä). Tarvittaessa lisää tarkempi kuvaus tyhjän rivin jälkeen.
Esimerkkejä hyvistä commit-viesteistä:
  • "Fix login bug"
  • "Add user profile page"
  • "Update README with installation instructions"
  • "Minor UI improvements"
Esimerkkejä huonoista commit-viesteistä:
  • "stuff" (liian epämääräinen)
  • "asdasd" (merkityksetön)
  • "final version" (ei kerro mitä muutettiin)
  • "added new feature but its still broken" (epäselvä)
  • "Fix login, update styles, change database schema, remove old tests, add new feature" (liian monta asiaa yhdessä)

Hyvä commit-viesti kertoo mitä muutettiin ja miksi. Kirjoita se niin, että myös viikon päästä ymmärrät viestin sisällön.
