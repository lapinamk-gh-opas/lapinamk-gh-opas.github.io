GitHubin turvallinen käyttö

GitHub on yksi maailman käytetyimmistä versionhallintapalveluista, ja sitä käytetään laajasti sekä avoimen lähdekoodin projekteissa että kaupallisessa ohjelmistokehityksessä. Suuren käyttäjä- ja projektimäärän vuoksi GitHub on houkutteleva kohde hyökkäyksille ja tietomurroille. Projekteissa voi olla arkaluontoisia tietoja, kuten salasanoja, API-avaimia, konfiguraatioita tai yhteystietoja, joten jokaisen kehittäjän on tärkeää osata huolehtia tietoturvasta ja yksityisyydestä.

Tässä kappaleessa käsitellään GitHubin turvallista käyttöä ja annetaan vinkkejä tilisi ja projektiesi suojaamiseen.

Käytä vahvaa tunnistautumista

GitHub-tilin suojaus on ensisijaisen tärkeää. Käytä aina vahvaa tunnistautumista kirjautuessasi GitHubiin. Huolehdi että sinulla on ainakin:

  • Vahva ja uniikki salasana
  • Kaksivaiheinen tunnistautuminen (2FA) käytössä

GitHub tukee myös muita turvallisia kirjautumistapoja, kuten autentikointisovelluksia, turva-avaimia (security key), sekä komentorivillä käytettäviä Personal Access Tokeneita ja SSH-avaimia.

Pidä ohjelmistot ja työkalut ajan tasalla

Varmista, että käytät aina ajantasaisia versioita Gitistä, IDE:stä, sekä muista kehitystyökaluista. Päivitykset sisältävät usein tietoturvakorjauksia, jotka suojaavat sinua tunnetuilta haavoittuvuuksilta. Muista myös varmistaa, että käytössäsi olevat kolmannen osapuolen laajennukset ja työkalut ovat luotettavia ja päivitettyjä.

Voit tarkistaa Gitin version komennolla git --version ja tarvittaessa päivittää sen Gitin virallisilta sivuilta, käyttöjärjestelmäsi pakettihallinnasta tai komennolla git update-git-for-windows / vastaavalla.

Huolehdi repositorion asetuksista

Varmista, että repositorion yksityisyysasetukset vastaavat projektin tarpeita. Käytä yksityisiä repositorioita, kun työskentelet luottamuksellisten projektien parissa. Huolehdi myös käyttöoikeuksista: anna tiimin jäsenille vain ne oikeudet (least privilege), joita he tarvitsevat työnsä tekemiseen, ja poista tarpeettomat oikeudet säännöllisesti.

Käytä GitHubin tietoturvaominaisuuksia

GitHub tarjoaa myös useita tietoturvatyökaluja, jotka auttavat suojaamaan projektejasi. Esimerkiksi:

  • Dependabot - tarkistaa riippuvuudet haavoittuvuuksien varalta ja ehdottaa päivityksiä
  • Code Scanning (CodeQL) - analysoi koodia mahdollisten virheiden ja turvallisuusongelmien varalta

Ota nämä työkalut käyttöön repositorioissasi ja seuraa niiden ilmoituksia säännöllisesti oppiaksesi lisää projektisi turvallisuudesta.

Löydät GitHubin tietoturva- ja analytiikkaominaisuudet repositorion asetuksista kohdasta Settings → Advanced Security. Kokeile niitä rohkeasti!

Ole tarkkana, mitä tietoja jaat

Älä lisää versiohallintaan salaisia tietoja, kuten salasanoja, API-avaimia tai henkilötietoja. Git tallentaa kaiken historiaansa, joten luottamuksellista tietoa ei pidä tallentaa edes väliaikaisesti.

Seuraavalla sivulla käydään läpi käytännön keinot, joilla varmistat, etteivät nämä tiedot päädy repositorioon vahingossa.
