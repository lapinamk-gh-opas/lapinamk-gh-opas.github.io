GitHubin turvallinen käyttö

GitHub on maailman laajimmin käytetty versionhallintapalvelu, ja suurin osa ohjelmistoprojekteista hyödyntää sitä tavalla tai toisella. Tämä tekee siitä myös houkuttelevan kohteen hyökkäyksille ja tietomurroille. GitHub-repositorioissa voi olla arkaluontoisia tietoja, kuten salasanoja, API-avaimia, konfiguraatioita tai yhteystietoja. Siksi jokaisen kehittäjän on tärkeää osata huolehtia tietoturvasta ja yksityisyydestä.

Käytä vahvaa tunnistautumista

Yksi tehokkaimmista keinoista suojata oma GitHub-tili on käyttää:

  • vahvaa, uniikkia salasanaa
  • ja ottaa käyttöön kaksivaiheinen tunnistus (2FA)

Tämä suojaa tiliäsi, vaikka salasana vuotaisi.

Kaksivaiheisen tunnistautumisen lisäksi GitHub tukee myös muita turvallisia kirjautumistapoja, kuten autentikaatiosovelluksia, turva-avaimia (security key), sekä komentorivillä käytettäviä Personal Access Tokeneita ja SSH-avaimia.

Pidä ohjelmistot ja työkalut ajan tasalla

Varmista, että käytät aina ajantasaisia versioita Gitistä, GitHubin työpöytäsovelluksesta ja muista kehitystyökaluista. Päivitykset sisältävät usein tietoturvakorjauksia, jotka suojaavat sinua tunnetuilta haavoittuvuuksilta.

Voit tarkistaa Gitin version komennolla git --version ja päivittää sen Gitin viralliselta sivulta, käyttöjärjestelmäsi pakettihallinnasta tai komennolla git update-git-for-windows/vastaavalla.

Lisäksi, jos käytät kolmannen osapuolen työkaluja tai laajennuksia, tarkista niiden luotettavuus ja päivitä ne säännöllisesti.

Huolehdi repositorion asetuksista

Varmista, että repositorion yksityisyysasetukset vastaavat projektin tarpeita. Käytä yksityisiä repositorioita, kun työskentelet luottamuksellisten projektien parissa.

Huolehdi myös käyttöoikeuksista. Anna tiimin jäsenille vain ne oikeudet, joita he tarvitsevat työssään, ja poista tarpeettomat käyttöoikeudet säännöllisesti.

Ole tarkkana, mitä tietoja jaat

Älä koskaan lisää versiohallintaan salaisia tietoja, kuten salasanoja, tunnuksia tai henkilötietoja. Muista, että kaikki, mitä tallennat Gitin historiaan, säilyy siellä pysyvästi.

Jos projekti voi joskus päätyä julkiseksi, vältä lisäämästä sinne mitään luottamuksellista. Git tallentaa kaiken, myös sen, minkä olet myöhemmin poistanut.

Käytä GitHubin tietoturvaominaisuuksia

GitHub tarjoaa myös useita työkaluja, jotka auttavat suojaamaan projektejasi. Esimerkiksi:

  • Dependabot, joka tarkistaa riippuvuudet haavoittuvuuksien varalta ja ehdottaa päivityksiä
  • Code Scanning (CodeQL), joka analysoi koodia mahdollisten virheiden ja turvallisuusongelmien varalta

Voit ottaa nämä työkalut käyttöön repositorioissasi ja seurata niiden ilmoituksia säännöllisesti.

Vinkki: Löydät GitHubin tietoturva- ja analytiikkaominaisuudet repositorion asetuksista kohdasta Settings → Advanced Security. Kokeile niitä rohkeasti!
