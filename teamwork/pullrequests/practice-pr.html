Pull Request

Tiimityössä pull request varmistaa, että muutokset tarkistetaan ennen yhdistämistä päähaaraan. Tässä harjoituksessa kerrataan sen luomista ja muutosten hyväksyntää.

Harjoitus 4 – Pull Requestin luominen
  1. Avaa edellisessä harjoituksessa käyttämäsi repositorio
  2. Luo uusi kehityshaara päähaarasta selkeällä nimellä (esim. pull-request)
  3. Muokkaa index.html-tiedostoa lisäämällä uusi <p>-kappale, joka kertoo mitä olet oppinut 
    
<p>Osaan tehdä pull requestin.</p>

    Tallenna tiedosto

  4. Commitoi muutos selkeällä commit-viestillä
  5. Julkaise haara GitHubiin
  6. Avaa GitHub ja tarkista, että uusi haara ja commit näkyvät repositoriossa
  7. Tee pull request kehityshaaran yhdistämiseksi päähaaraan
  8. Kirjoita pull requestille otsikko ja lyhyt selitys tekemistäsi muutoksista, sekä aseta itsesi tarkistajaksi
  9. Tarkista tekemäsi muutokset pull requestin files changed-näkymässä
  10. Lisää kommentti pull requestiin
  11. Hyväksy ja yhdistä pull request päähaaraan
  12. Poista haara yhdistämisen jälkeen, jos se ei ole enää tarpeellinen
  13. Tarkista, että päähaaran sisältö on päivittynyt ja muutokset näkyvät oikein

Jos sait harjoituksen tehtyä, hienoa! Olet oppinut käyttämään pull requesteja GitHubissa!

Seuraavaksi tutustumme tarkemmin merge-konflikteihin ja niiden käsittelyyn.
