Git ja GitHub

Git ja GitHub eivät ole sama asia.

  • Git = versionhallintatyökalu, jota käytät omalla tietokoneellasi
  • GitHub = pilvipohjainen palvelu, joka hyödyntää Gitiä taustalla

Tällä sivulla tutustutaan niihin paremmin.

Versionhallinta

Versionhallinta on järjestelmä, jolla seurataan ja hallitaan tiedostoihin tehtyjä muutoksia. Sen avulla voidaan palauttaa aiempia versioita, tarkastella projektin historiaa ja työskennellä usean ihmisen kanssa samoissa tiedostoissa yhtä aikaa.

Johdanto Git:iin

Git on erittäin suosittu versionhallintajärjestelmä, jonka loi Linuxin kehittäjä Linus Torvalds vuonna 2005. Se on avoimen lähdekoodin projekti ja täysin ilmainen.

Git toimii paikallisesti omalla tietokoneella, eikä sen käyttö vaadi internet-yhteyttä. Git sopii ohjelmistokehityksen lisäksi myös muihin projekteihin, joissa halutaan hallita tiedostojen muutoksia, kuten esim. animointiin, dokumentointiin tai kirjoittamiseen.

Gitin keskeiset käsitteet lyhyesti

  • Repository (repo) - Kansio, jossa Git seuraa projektiasi ja sen historiaa
  • Clone - Tee kopio etäreposta omalle tietokoneellesi
  • Commit - Tallenna tehdyt muutokset
  • Branch - Luo haara erillisten versioiden tai ominaisuuksien kehittämistä varten
  • Merge - Yhdistä muutokset eri haaroista (branch)
  • Pull - Tuo viimeisimmät muutokset etäreposta omalle tietokoneellesi
  • Push - Vie omat muutoksesi etärepoon

Huom! Suurin osa Gitin toiminnoista tapahtuu omalla tietokoneellasi. Vain Push ja Pull yhdistävät etäpalvelimeen kuten GitHubiin.
