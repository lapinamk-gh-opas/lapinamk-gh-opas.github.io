Git-tiedoston tilat (git status)

Tiedoston tila kertoo missä vaiheessa sen muutokset ovat. Tilat näkyvät esimerkiksi VS Code Explorerin hakemistorakenteessa tilakoristeilla (status decorations), eli eri väreillä ja lyhenteillä, sekä git status -komennon tulosteessa.

Komento git status näyttää aina ajantasaisen yhteenvedon repositorion tiedostoista poikkeavien tiedostojen tilanteesta.

Uusi tiedosto näkyy ensin Untracked (U)-tilassa, kunnes lisäät sen komennolla git add. Tällöin se siirtyy staging-alueelle, muuttuu tilaan Added (A) ja on valmiina seuraavaan committiin. Jos muokkaat jo seurattua tiedostoa se näkyy tilassa Modified (M), kunnes committaat muutokset repositorioon.

Kun commit on tehty, tiedostojen tilakoristeet katoavat, eikä commitattuja tiedostoja enää näy git status -komennolla.

Taulukossa on esitetty yleisimmät tilat, niiden värit, sekä lyhyt kuvaus siitä, mitä tila tarkoittaa käytännössä.

Tämä tarjoaa visuaalisen keinon hahmottaa projektin tilanne yhdellä silmäyksellä ja seurata helposti, mitä tiedostoille on tapahtunut.
