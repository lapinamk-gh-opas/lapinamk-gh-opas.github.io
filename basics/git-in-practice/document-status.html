Git-tiedoston tilat (VS Code -näkymä)

Gitissä tiedostojen tilat kertovat, missä vaiheessa muutosten hallinta on. Kun lisäät, muokkaat tai poistat tiedostoja, Git seuraa niitä eri tilojen avulla.

GitLens (suositus): VS Coden perusnäkymässä näkyy usein vain M (Modified). GitLens-lisäosa näyttää selkeästi myös U (Untracked), A (Added), D (Deleted), R (Renamed), C (Copied) ja muut tilat, sekä tarjoaa rivikohtaisen blame-tiedon, tiedoston/rivin historian ja nopeat diff-vertailut. Se helpottaa Gitin käyttöä VS Codessa merkittävästi.

Asenna: avaa Extensions (Ctrl+Shift+X) → etsi “GitLens — Git supercharged” → Install.

Taulukossa on esitetty yleisimmät tilat, niiden värit ja lyhenne Visual Studio Codessa, sekä lyhyt kuvaus siitä, mitä tila tarkoittaa käytännössä.

Tila Väri Selitys
Untracked (U) Vihreä Uusi tiedosto, jota Git ei vielä seuraa. Lisää versiohistoriaan komennolla git add.
Unmodified (-) Ei väriä Ei muutoksia edellisen commitin jälkeen. Synkassa repositorion kanssa.
Modified (M) Keltainen Tiedostoa on muokattu, mutta muutoksia ei ole lisätty staging-alueelle.
Added (A) Vihreä Muutokset on lisätty stagingiin komennolla git add ja ovat valmiina commitille.
Deleted (D) Yliviivattu Tiedosto on poistettu.
Committed (-) Ei väriä Muutokset on tallennettu repositorioon (paikallisesti). Pushaa GitHubiin komennolla git push.

Tämä tarjoaa visuaalisen keinon hahmottaa projektin tilanne yhdellä silmäyksellä ja nähdä nopeasti, mitkä tiedostot vaativat vielä toimenpiteitä ja mitkä ovat jo valmiina tallennettavaksi versiohistoriaan.
