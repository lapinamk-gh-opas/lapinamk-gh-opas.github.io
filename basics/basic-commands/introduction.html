Git - Keskeiset komennot

Gitin käyttäminen komentokehotteesta on nopea ja tehokas tapa hallita versionhallintaa. Kun opit peruskomennot, työskentelet sujuvasti IDE:stä riippumatta ja saat täyden hallinnan Git-toimintoihin.

Tässä on yleisimmät ja hyödyllisimmät Git-komennot, jotka jokaisen kehittäjän tulisi ymmärtää. Samat komennot löytyvät myös sivupalkin komennot -muistilistasta, joten sinun ei tarvitse opetella niitä ulkoa.

Sivupalkista löydät lisäksi muistilistan yleisimmistä workflow-työnkuluista, jotka auttavat muistamaan, missä järjestyksessä komennot kannattaa ajaa.

On tärkeä myös tiedostaa, että Gitissä samaan lopputulokseen pääsee usein monella eri tavalla. Valitse tilanteeseen ja sinulle sopivin. Yhtä ainoaa oikeaa tapaa ei ole.

Kansionhallinta

Siirry kansioon - cd Projekti

Vaihtaa nykyisen työskentelykansion toiseen. Aja tämä ennen Git-komentoja, jotta olet oikeassa projektikansiossa.

Siirry yksi taso ylemmäs - cd ..

Liikkuu hakemistopuussa ylöspäin. Eli esimerkiksi kansiosta Projekti kansioon, jossa Projekti sijaitsee.

Näytä kansion sisältö - dir / ls

Listaa nykyisen kansion tiedostot ja kansiot (Windows: dir, macOS/Linux: ls). Varmistat nopeasti mitä kansiossa on.

Luo uusi kansio - mkdir harjoitus-1

Luo tyhjän kansion esimerkiksi uutta harjoitusta tai projektia varten. Siirry sitten kansioon komennolla cd harjoitus-1

Älä hätkähdä komentojen määrää! Käymme ne läpi tapauskohtaisesti esimerkein ja harjoituksin.

Git peruskomennot

Alusta uusi Git-repositorio - git init

Käynnistää versionhallinnan nykyisessä kansiossa luomalla .git-hakemiston. Aja vain projektin juuressa.

Kloonaa olemassa oleva repositorio - git clone https://github.com/user/projekti.git

Lataa etärepositorion sisällön paikallisesti ja luo repositorio uuden kansion tietokoneellesi.

Yhdistä paikallinen repositorio etärepositorioon - git remote add origin <url>

Linkittää paikallisen repositorion etärepositorioon (esim. GitHubiin). Tarvitaan vain kerran per repositorio.

Näytä muutosten tila - git status

Näyttää uudet, muokatut ja stagingiin lisätyt tiedostot, sekä nykyisen haaran. Ensimmäinen komento, kun mietit missä mennään. Tiivis versio git status -sb

Lisää tiedosto stagingiin - git add index.html

Valmistelee yksittäisen tiedoston seuraavaan committiin. Toista useille tiedostoille tarpeen mukaan.

Lisää kaikki muutokset stagingiin - git add .

Kerää kaikki kansion ja alikansioiden muutokset kerralla stagingiin. Nopea tapa valmistella commit.

  • Käytä komentoa git add . varoen. Varsinkin jos projekti sisältää arkaluontoista tietoa. Harkitse myös .gitignore -tiedoston lisäämistä projektiin. Lisää tietoa gitignoresta luvussa 2.5 Yksityisyys ja tietoturva.
Tallenna muutokset (commit) - git commit -m "commit-viesti"

Kirjaa stagingissa olevat muutokset paikalliseen historiaan. Pidä commit-viesti lyhyenä, mutta kuvaavana. Lisää commit-viesteistä luvussa 2.1 Selkeät commit-viestit.

Git-komentojen lipuilla voidaan muokata komennon toimintaa ja tarkkuutta. Jokaisella lipulla on oma tarkoituksensa.

Liput ovat usein lyhenteitä tai symboleita, jotka helpottavat komentojen kirjoittamista, kuten -m, -u, --soft, --hard, jne.

  • git commit -m "viesti" - lipun -m avulla voit liittää commit-viestin suoraan komentoon. Ilman -m -lippua komennossa, Git avaa tekstieditorin, johon voit kirjoittaa commit viestin.
  • git push -u origin <haara> - lisäämällä -u asetat seurannan etähaaraan, jolloin jatkossa riittää pelkkä git push/git pull
  • git checkout -b <haara> - lisäämällä -b luot uuden haaran ja siirryt siihen. Pelkkä git branch <haara> luo uuden haaran, muttei siirry siihen.
  • git log --oneline - lisäämällä --oneline saat tiiviin commit-lokin yhdellä rivillä per commit.
Tallenna keskeneräiset muutokset väliaikaisesti (stash) - git stash

Säilyttää tekemäsi muutokset tilapäisesti ja palauttaa työskentelyhakemiston viimeisimpään commit-tilaan. Hyödyllinen, kun haluat vaihtaa haaraa tai päivittää projektin ilman, että keskeneräiset muutokset häiritsevät. Palauta muutokset komennolla git stash pop

Tähän asti esitellyt komennot on Perusteet -luvun komentoja. Voit keskittyä aluksi niihin.

Git-haarojen hallinta

Näytä kaikki haarat - git branch

Listaa paikalliset haarat. Nykyinen haara on merkitty tähdellä (*).

Luo ja siirry haaraan - git checkout -b haaran-nimi

Luo uuden kehityshaaran ja vaihtaa siihen. Vaihtoehtoisesti git switch -c haaran-nimi

Siirry olemassa olevaan haaraan - git checkout haaran-nimi

Vaihda kehitystyö toiseen haaraan. Varmista, että keskeneräiset muutokset on commitoitu tai stashattu ennen vaihtoa. Vaihtoehtoisesti git switch haaran-nimi

Hae ja yhdistä tietyn etärepositorion haaran muutokset - git pull origin haaran-nimi

Hakee uusimmat commitit etärepositorion haarasta ja yhdistää ne nykyiseen haaraan.

Puske haaran sisältö ja aseta seuranta - git push -u origin haaran-nimi

Komento luo vastaavan etähaaran, lähettää nykyisen haaran sisällön ja asettaa seurannan (upstream). Tämän jälkeen jatkossa riittää pelkkä git push

Seuranta asetetaan push-komennon lipulla -u (tai --set-upstream). Tämä määrittää paikalliselle haaralle oletusvastinparin etärepositoriossa, johon muutokset lähetetään ja josta ne haetaan jatkossa.
Hae ja yhdistä etärepositorion muutokset - git pull

Hakee uusimmat commitit etärepositorion vastaavasta haarasta ja yhdistää ne nykyiseen haaraan.

Puske muutokset etärepositorioon - git push

Lähettää nykyisen haaran commitit etärepositorion vastaavaan haaraan.

Yhdistä haara nykyiseen - git merge haaran-nimi

Yhdistää komennossa mainitun haaran nykyiseen haaraan. Varmista, että työpuu on puhdas ennen yhdistämistä, eli kaikki muutokset on commitoitu ja staging on tyhjä.

Poista haara - git branch -d haaran-nimi

Poistaa paikallisen haaran. Varmista, että kaikki muutokset on yhdistetty ennen poistamista. Poista etähaara komennolla git push -d origin haaran-nimi

Lisäkomennot

Näytä erot muutosten välillä - git diff

Vertaa työpuuta viimeisimpään commit-tilaan. Tarkista stagingiin lisätyt erot lisäämällä --staged

Palauta tiedosto viime commitin tilaan - git restore index.html

Poistaa paikalliset muutokset valitusta tiedostosta (ei muuta commit-historiaa). Huom! Committamattomat muutokset häviävät.

Peru viimeisin commit - git reset HEAD~

Siirtää haaran taaksepäin committia edeltäneeseen tilaan ja jättää muutokset työpuuhun. Käytä vain, jos commiteja ei ole pushattu etärepositorioon. Tiimiprojekteissa käytä mielummin git revert <commit>

Näytä commit-historia - git log

Listaa commitit nykyisessä haarassa uusimmasta vanhimpaan. Lisää --oneline tiiviimpään näkymään.

Säännöllinen käyttö tekee komennoista rutiinia ja Gitin käytöstä sujuvaa. Kappaleiden edetessä voit aina palata tähän listaan, tai sivupalkin muistilistaan kertaamaan komentoja ja niiden käyttötarkoituksia.
