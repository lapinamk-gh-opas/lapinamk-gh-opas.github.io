Gitin käyttö Visual Studio Codessa

Tällä sivulla käsitellään Gitin käyttöä Microsoftin kehittämällä koodieditorilla Visual Studio Codella (tai lyhyemmin VS Codella ).

Gitin ja GitHubin käyttöönottoon Visual Studio Codessa ei tarvita yleensä muita toimenpiteitä kuin ne, jotka on tehty osiossa 1.4 Käyttöönotto . Kirjautumalla kuitenkin sisään omaan GitHub-tiliisi saat käyttöösi useita lisäominaisuuksia, kuten VS Coden asetusten varmuuskopioinnin, GitHub Copilotin ja muita GitHub-liitännäisiä.

Kirjautuminen tapahtuu seuraavasti
  1. Klikkaa vasemmalta alhaalta Accounts kuvaketta
    Valitse Backup and Synch Settings...
    Valitse mitä asetuksia haluat synkronoida ja klikkaa Sign in
  2. Valitse kirjautumiseen käytettävä tili klikkaamalla Sign in with GitHub
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjautunut sisään, voit siirtyä suoraan kohtaan 6. Käytettävän tilin valinta .
  3. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti ja salasana
    Klikkaa Sign in
  4. Anna kaksivaiheisen tunnistautumisen avainkoodi tunnistautumissovelluksestasi
    Klikkaa Verify
  5. Lisää halutessasi passkey tai klikkaa Ask me later
  6. Valitse käytettävä tili klikkaamalla Continue käyttäjänimen vierestä

    Mikäli sinulla oli GitHub avoinna selaimessa ja olit jo valmiiksi kirjautuneena sinne sisään ja jos sinulla on kaksivaiheinen tunnistautuminen käytössä saattaa GitHub tämän vaiheen jälkeen pyytää vielä syöttämään tunnistautumissovelluksen avainkoodin (kts. kohta 4).

  7. Anna VS Codelle lupa päästä GitHub-tilillesi klikkaamalla Authorize Visuals-Studio-Code
  8. Siirry takaisin VS Codeen klikkaamalla Avaa Visuals-Studio-Code
  9. Halutessasi ota Copilot käyttöön ja asenna tarvittavat liitännäiset
    • Klikkaa ensin Finnish Setup
    • Valitse sitten Set up Copilot

Osiossa luodaan edellisen sivun malliin editorin avulla jälleen tiedosto ja paikallinen repositorio sekä viedään se samaan, edellisessä osiossakin käytettyyn, etärepositorioon, mutta käytetään tälle osiolle omaa haaraansa. Gitin käyttöön VS Coden terminaalista voit hyödyntää edellisen sivun ohjeita.

Huomiona mainittakoon, että VS Code tarjoaa mahdollisuuksia tehdä Git toimintoja usealla eri tavalla. Jotkin toiminnoista voi tehdä mahdollisesti myös toisella tapaa, ehkäpä jopa paremmin tai tehokkaammin kuin mitä ohjeessa on esitetty.

Tiedoston luonti ja repojen yhdistäminen VS Codessa

  1. Luo uusi kansio testausta varten
    • Klikkaa Open Folder
    • Navigoi haluamaasi kansioon ja klikkaa Uusi kansio
    • Anna kansiolle nimi ja klikkaa Valitse kansio
    • Jos tiedät, että kansio on luotettava (esim. oma projektisi), valitse Yes, I trust the authors .
    • Jos et ole varma kansion tai projektin sisällöstä tai luotettavuudesta, voit avata kansion “Restricted Mode” -tilassa, jolloin mm. laajennukset ja koodin suoritus estetään.
  2. Luo paikallinen repositorio
    • Avaa Source Control-valikko klikkaamalla sivupalkin kuvakkeesta tai näppäimillä CTRL + SHIFT + G . Klikkaa Source Control kuvaketta sivuvalikosta.
    • Valitse avautuneesta valikosta Initialize Repository
  3. Yhdistä paikallinen repositorio etärepositorioon
    • Klikkaa kolmea pistettä luodun repositorion vierestä ja valitse Remote-valikon alta valinta Add Remote...
    • Anna yläpalkin kenttään etärepositorion osoite https://github.com/käyttäjänimi/repositorion-nimi.git
    • Klikkaa Add remote from URL
    • Anna yläpalkin kenttään etärepositorion nimi esim. origin (gitin oletus) ja paina Enter
  4. Tuo (Pull) etärepositorion tila paikalliseen repositorioon
    • Mikäli käytät uutta tyhjää etärepositoriota jätä tämä vaihe välistä ja siirry kohtaan 5. Luo uusi tiedosto testausta varten .
    • Klikkaa kolmea pistettä ja valitse Pull, Push-valikon alta valinta Pull from...
    • Valitse haara jonka sisällön haluat tuoda
    • origin/HEAD tarkoittaa aina etärepositorion oletushaara (Default)
    • Tässä tapauksessa repositorion oletus haara on nimeltään origin/main
    • Eli molemmat vaihtoehdot ovat siis tällä hetkellä tämän repositorion kohdalla yksi ja sama haara
  5. Luo uusi tiedosto testausta varten
    • Avaa Explorer-valikko klikkaamalla sivupalkin kuvakkeesta tai näppäimillä CTRL + SHIFT + E
    • Kansion nimen vierestä uusi tiedosto kuvaketta
    • Anna tiedostolle nimi esim. tiedosto-vs-codella.md + enter
    • Lisää tiedostolle sopiva sisältö ja tallenna pikanäppäimellä CTRL + S
    • Valitse README.md, muuta sen sisältöä ja tallenna
    • Avaa Source Control -valikko klikkaamalla sivupalkin kuvakkeesta tai näppäimillä CTRL + SHIFT + G
    • Klikkaa Changes -valikon alta README.MD tiedostoa
    • Aukeaa ruutu README.md (working tree) jonka avulla voit tarkastella muutoksia tiedoston edellisen commitin tilaan verrattuna
    • Ikkunassa näkyy tehdyt muutokset vihreällä ja aikaisempi koodi punaisella
  6. Luo repositoriolle uusi haara (branch)
    • Klikkaa kolmea pistettä ja valitse Branch-valikon alta valinta Create Branch...
    • Anna yläpalkin kenttään uuden haaran nimi esim. vs-code-haara ja paina Enter
    • Teoriaa Gitin haaroihin (branches) liittyen käydään läpi tarkemmin osiossa 2.1 Tiimityö ja haarat .
  7. Lisää muutokset ja uusi tiedosto välialueelle (staging) ja tee commit
    • Klikkaa kolmea pistettä ja valitse Changes -valikon alta valinta Stage All Changes . Tämä vastaa komentoa git add .
    • Käytä komentoa git add . varoen. Varsinkin jos projekti sisältää arkaluontoista tietoa. Harkitse myös .gitignore -tiedoston lisäämistä projektiin. Lisää tietoa gitignoresta luvussa 2.5 Yksityisyys ja tietoturva .
    • Anna message kenttään commit-viesti esim. ensimmäinen commit vs codella
    • Klikkaa Commit painiketta
    • Jätä message kenttä tyhjäksi ja paina Commit painiketta
    • Aukeaa uusi ikkuna COMMIT_EDITMSG
    • Lisää commit viesti ensimmäiselle riville ja tallenna tiedosto CTRL + S
    • Hyväksy viesti klikkaamalla oikealta ylhäältä Accept Commit Message painiketta
  8. Vie (push) muutokset etärepositorioon
    • Klikkaa kolmea pistettä ja valitse Push
    • VS Code ilmoittaa, että etärepositoriossa ei ole tämän nimistä haaraa
    • Valitsemalla OK voit luoda haaran etärepositorioon
    • Vaihtoehtoisesti voit painaa Publish Branch -painiketta, joka luo uuden haaran etärepositorioon ja vie muutokset sinne samalla
  9. Tarkista luotu haara ja tuodut muutokset GitHubissa
    • Siirry GitHubiin ja avaa repositorio
    • Klikkaa haaran valintaa jossa lukee main
    • Valitse luotu haara
    • Tarkastele luodun haaran sisältöä
