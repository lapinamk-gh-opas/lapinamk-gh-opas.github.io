Gitin käyttö PyCharmilla

Tällä sivulla käsitellään Gitin käyttöä PyCharmilla. Haitari elementistä voit katsoa tarvittaessa ohjeet Gitin ja GitHubin käyttöönottoon editorille. Muutoin osion ohjeissa käytetään uutta repositoriota, cloonataan se paikalliseksi repositorioksi IDE:llä. Luodaan editorin avulla jälleen tiedostot (oletus main.py ja README.md) ja viedään ne etärepositorion päähaaraan.

Huomiona mainittakoon, että myös PyCharm tarjoaa mahdollisuuksia tehdä Git toimintoja usealla eri tavalla. Jotkin toiminnoista voi tehdä mahdollisesti myös toisella tapaa, ehkäpä jopa paremmin sekä tehokkaammin kuin mitä ohjeessa on esitetty.

Gitin ja GitHubin käyttöönottamiseksi PyCharmissa käy läpi, 1.4 käyttöönotto osiossa tehtyjen toimien lisäksi, seuraavat asetukset.

  1. Klikkaa vasemmalta alhaalta Ratas kuvaketta ja valitse Settings
  2. Valitse asetusvalikon vasemmalta puolelta listasta Version Control ja sen alta Git
    • Testaa, että PyCharm löytää Gitin painamalla Test nappia.
    • Mikäli Git löytyy näykyy sen version numero polun alla.
    • Mikäli Gitiä ei löytynyt anna PyCharmille polku johon olet Gitin asentanut.
  3. Valitse asetusvalikon vasemmalta puolelta listasta Version Controlin alta GitHub
    • Yhdistä GitHub tilisi PyCharmiin klikkaamalla Add account... tekstiä.
    • Valitse Log in via GitHub...

    GitHub-tilin yhdistäminen editoriin voi hieman erota tässä esitetystä, mikäli olet jo antanut oikeudet jollekin toiselle JetBrainsin editorille käyttää tilisi sisältöä.

  4. Avautuu selain ikkuna ja klikkaa Authorize in GitHub nappia
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjatunut sisään, voit siirtyä suoraan kohtaan: 7. Luvananto PyCharmille.
  5. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti ja salasana
    Klikkaa Sign in
  6. Anna kaksivaiheisen tunnistautumisen avainkoodi tunnistautumissovelluksestasi ja klikkaa Verify
  7. Anna PyCharmille lupa päästä GitHub-tilillesi klikkaamalla Authorize JetBrains

    Mikäli sinulla oli GitHub avoinna selaimessa ja olit jo valmiiksi kirjautuneena sinne sisään ja jos sinulla on kaksivaiheinen tunnistautuminen käytössä saattaa GitHub tämän vaiheen jälkeen pyytää vielä syöttämään tunnistautumissovelluksen avainkoodin (kts. kohta 6).

  8. Tulee ilmoitus, että olet sallinut PyCharmille pääsyn GitHub-tilillesi ja voit sulkea selainikkunan.
  9. Myös PyCharm näyttää GitHub asetusvälilehden alla lisäämäsi tilin.
    Sulje asetusikkuna klikkaamalla OK nappia.
  1. Luo uusi repositorio gitin testaamista varten osan 1.4 Käyttöönotto/GitHub-repositorion luominen ohjeen mukaan
  2. Kloonaa luomasi etärepositorio PyCharmiin
    • Klikkaa projects valikossa Clone Repository joko Welcome to PyCharm tekstin alta tai mikäli sinulla on aikaisempia projekteja niin niiden yläpuolelta oikeanpuolimmaisesta napista
  3. Anna kloonattava repositorio.
    • Repository URL valikossa anna URL-kenttään etärepositorion osoite.
    • Muuta halutessasi kohde kansio Directory-kenttään.
    • Klikkaa Clone nappia.
    • Jos tiedät, että kansio on luotettava (esim. oma projektisi), valitse Trust Project.
    • Jos et ole varma kansion tai projektin sisällöstä ja luotettavuudesta, voit avata kansion klikkaamalla Preview in Safe Mode-nappia, jolloin voit tarkastella sisältö ennen siihen luottamista.
  4. PyCharmin luoma main.py tiedosto
    • PyCharm loi projektille main.py ja kysyy haluatko lisätä tiedoston versionhallintaan. Voi lisätä sen nyt painamalla Add nappia tai klikata Cancel nappia ja lisätä sen myöhemmin esitellyllä tavalla
  5. README.md tiedosnton lisääminen
    • Valitse -ikonilla aukeavasta alasvetovalikosta File.
    • Anna lisättävälle tiedostolle nimi README.md ja paina Enter.
    • Voi taas lisätä tiedoston versionhallintaa kuten kohdassa 4 tai jättää sen myöhempään.
    • Lisää tiedostolle sopiva sisältö ja tallenna pikanäppäimellä CTRL + S
  6. Lisätään uudet tiedosto staging tilaan.
    • Klikkaa kansiota jossa tiedostot ovat.
    • Tämän jälkeen klikkaa ylävalikosta Main Menu kuvaketta.
    • Valitse päävilikosta välilehti Git, Sen alta Selected Directory ja klikkaa Add.
    • Tämä lisää kaikki kansion tiedostot staging tilaan.
    • Vaihtoehtoisesti voit tehdä lisäämisen pikanäppäimillä painamalla kansion valitsemisen jälkeen CTRL + ALT + A. Tai voi valita kansion sijaan yksittäisen tiedonston ja viedä sen staging tilaan, joko valikosta (Main Menu > Git > Current file > Add) tai pikanäppäimillä.
    • Tämä vastaa komentoa: git add .
    • Käytä komentoa git add . varoen.
      Jos olet lisännyt samaan kansioon jotain arkaluontoistatietoa sisältäviä tiedostoja, joita et halua lisätä repositioon, niin älä käytä git add . komentoa tai lisää repositorioon .gitignore -tiedosto, joka estää siihen nimettyjen tiedostojen lisäämisen repositorioon.
    • .gitignore:sta lisää tietoa osiossa: 2.5 yksityisyys ja tietoturva
    • Lisättyjen tiedostojen väri muuttuu punaisesta vihreäksi.
  7. Muutoksin commitoiminen, viestin lisääminen ja vienti etärepositorioon.
    • Avaa Commit valikko: ylävalikosta Main Menu > Git > Commit tai klikkaamalla sivupalkin Commit kuvaketta tai pikanäppäimillä CTRL + K.
    • Lisää commit-viesti tekstikenttään.
    • Paina Commit nappia commitoidaksesi muutokset paikalliseen repositorioon.
    • tai paina Commit and Push nappia commitoidaksesi muutokset paikallisesti ja viedäksesi muutoset etärepositorioon.
    • Mikäli teit vain commitin, voit viedä commitoidut muutokset etärepositorioon tekemällä pushin erikseen
    • Valitse Push joko yläpalkin versionhallinta pikavalikosta tai päävalikosta Main Menu > Git > Push tai pikanäppäimellä CTRL + SHIFT + K
    • Seuraava push ruutu avautuun niin Commit and Push napilla kuin erikseen pushatessa. Ruudussa näkyy pushattavan commit-viesti sekä pushattava haara ja tiedostot.
    • Klikkaa Push nappia viedäksesi muutokset etärepositorioon.
  8. Tuodut muutokset GitHubissa
    • Mene selaimella GitHubiin ja PyCharmiin yhdistämääsi repositorioon.
    • Nyt lisätyt tiedostot näkyvät GitHubissa
