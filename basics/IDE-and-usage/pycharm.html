Gitin käyttö PyCharmissa

Tällä sivulla käsitellään Gitin käyttöä PyCharmissa. Osion ohjeissa luodaan uusi etärepositorio GitHubiin, kloonataan se paikalliseksi repositorioksi PyCharmilla ja sen jälkeen luodaan tiedostot main.py ja README.md. Lopuksi tiedostot viedään etärepositorion päähaaraan.

Gitin ja GitHubin käyttöönottamiseksi PyCharmissa käy ensin läpi 1.4 Käyttöönotto osiossa käydyt asennusohjeet ja sen jälkeen seuraa alla olevia ohjeita.

  1. Klikkaa vasemmalta alhaalta Ratas kuvaketta ja valitse Settings
  2. Valitse asetusvalikon vasemmalta puolelta listasta Version Control ja sen alta Git
    • Testaa, että PyCharm löytää Gitin painamalla Test nappia
    • Mikäli Git löytyy näykyy sen versio numero polun alla
    • Mikäli Gitiä ei löytynyt anna PyCharmille polku johon olet Gitin asentanut
  3. Valitse asetusvalikon vasemmalta puolelta listasta Version Controlin alta GitHub
    • Yhdistä GitHub tilisi PyCharmiin klikkaamalla Add account... tekstiä
    • Valitse Log in via GitHub...

    GitHub-tilin yhdistäminen editoriin voi hieman erota tässä esitetystä, mikäli olet jo antanut oikeudet jollekin toiselle JetBrainsin editorille käyttää tilisi sisältöä.

  4. Avautuneessa ikkunassa klikkaa Authorize in GitHub nappia
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjautunut sisään, voit siirtyä suoraan kohtaan 7. Luvananto PyCharmille
  5. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti sekä salasana ja klikkaa Sign in painiketta
  6. Anna kaksivaiheisen tunnistautumisen avainkoodi tunnistautumissovelluksestasi ja klikkaa Verify
  7. Anna PyCharmille lupa päästä GitHub-tilillesi klikkaamalla Authorize JetBrains

    Jos olit kirjautuneena GitHubiin selaimessa ja käytät kaksivaiheista tunnistautumista, GitHub saattaa pyytää syöttämään vielä tunnistautumissovelluksen avainkoodin (katso kohta 6).

  8. Tarkista ilmoitus siitä, että olet sallinut PyCharmille pääsyn GitHub-tilillesi ja sulje selainikkuna.
  9. Tarkista myös PyCharmin asetuksista, että lisäämäsi GitHub-tili on näkyvissä ja sulje ikkuna

Huomiona mainittakoon, että PyCharm tarjoaa mahdollisuuksia tehdä Git toimintoja usealla eri tavalla. Jotkin toiminnoista voi tehdä mahdollisesti myös toisella tapaa, ehkäpä jopa paremmin tai tehokkaammin kuin mitä ohjeessa on esitetty.

Tiedoston luonti ja repositorioiden yhdistäminen PyCharmissa

  1. Luo uusi repositorio gitin testaamista varten 1.4 Käyttöönotto/GitHub-repositorion luominen ohjeen mukaan
  2. Kloonaa luomasi etärepositorio PyCharmiin
    • Klikkaa projects valikossa Clone Repository, joko Welcome to PyCharm tekstin alta tai mikäli sinulla on aikaisempia projekteja niin niiden yläpuolelta oikeanpuolimmaisesta napista.
  3. Anna kloonattava repositorion URL
    • Repository URL -valikossa anna URL -kenttään etärepositorion osoite
    • Muuta halutessasi kohde kansio Directory -kenttään
    • Klikkaa Clone nappia
    • Jos tiedät, että kansio on luotettava (esim. oma projektisi) valitse Trust Project
    • Jos et ole varma kansion tai projektin sisällöstä tai luotettavuudesta, voit avata kansion klikkaamalla Preview in Safe Mode-nappia, jolloin voit tarkastella sisältö ennen siihen luottamista.
  4. Tarkasta PyCharmin luoma main.py tiedosto
    • PyCharm loi projektille main.py -tiedoston ja kysyy haluatko lisätä tiedoston versionhallintaan. Lisää se painamalla Add -painiketta tai peru lisäys Cancel -napista ja lisää sen myöhemmin esitetyllä tavalla.
  5. Lisää README.md tiedosto
    • Valitse -ikonilla aukeavasta alasvetovalikosta File
    • Anna lisättävälle tiedostolle nimi README.md ja paina Enter
    • Voit taas valita lisäätkö tiedoston heti versionhallintaan, kuten kohdassa 4, vai jätätkö sen myöhempään.
    • Lisää tiedostolle sopiva sisältö, kuten
      Tämä on PyCharmilla luotu README.md tekstitiedosto testausta varten
      ja tallenna tiedosto CTRL + S
  6. Lisää uudet tiedostot välialueelle (staging)
    • Klikkaa kansiota jossa tiedostot ovat
    • Tämän jälkeen klikkaa ylävalikosta Main Menu kuvaketta
    • Valitse päävalikosta välilehti Git, sen alta Selected Directory ja klikkaa Add
    • Tämä vastaa komentoa git add . ja lisää kaikki kansion tiedostot välialueelle (staging)
    • Vaihtoehtoisesti voit tehdä lisäämisen pikanäppäimillä painamalla kansion valitsemisen jälkeen CTRL + ALT + A. Tai voi valita kansion sijaan yksittäisen tiedoston ja viedä sen välialueelle (staging), joko valikosta (Main Menu > Git > Current file > Add) tai pikanäppäimillä.
    • Käytä komentoa git add . varoen. Varsinkin jos projekti sisältää arkaluontoista tietoa. Harkitse myös .gitignore -tiedoston lisäämistä projektiin. Lisää tietoa gitignoresta luvussa 2.5 Yksityisyys ja tietoturva.
    • Lisättyjen tiedostojen väri muuttuu punaisesta vihreäksi
  7. Tee commit ja vie etärepositorioon
    • Avaa Commit valikko ylävalikosta Main Menu > Git > Commit tai klikkaamalla sivupalkin Commit kuvaketta tai pikanäppäimillä CTRL + K
    • Lisää commit-viesti tekstikenttään
    • Paina Commit nappia commitoidaksesi muutokset paikalliseen repositorioon
    • Vaihtoehtoisesti paina Commit and Push nappia commitoidaksesi muutokset paikallisesti ja viedäksesi muutokset etärepositorioon
    • Mikäli teit vain commitin, voit viedä commitatut muutokset etärepositorioon tekemällä pushin erikseen
    • Valitse Push joko yläpalkin versionhallinta pikavalikosta tai päävalikosta Main Menu > Git > Push tai pikanäppäimellä CTRL + SHIFT + K
    • Teitpä pushin millä tavalla tahansa, avautuu seuraavaksi viimeistelyikkuna, jossa näkyvät commitin viesti, haara ja siihen sisältyvät tiedostot
    • Klikkaa Push nappia viedäksesi muutokset etärepositorioon
  8. Tarkista tuodut muutokset GitHubissa
    • Siirry selaimessa kyseiseen repositorioon
    • Tarkista PyCharmissa lisäämäsi tiedostot
