Gitin käyttö GitHub Desktopilla

Tällä sivulla käsitellään Gitin käyttöä GitHub Desktopin avulla. GitHub Desktop ei ole koodieditori, joten sen kautta ei voi muokata tiedostoja. Työkalu on luotu varta vasten helpottamaan Gitin ja GitHubin käyttöä graafisella käyttöliittymällä.

GitHub Desktopilla voit hallita Git-toimintoja, kuten tiedostojen lisäämistä, haarojen luomista ja yhdistämistä sekä muutosten viemistä ja tuomista.

GitHub Desktopin käyttöönottamiseksi käy ensin läpi 1.4 Käyttöönotto osiossa käydyt asennusohjeet ja sen jälkeen seuraa alla olevia ohjeita.

  1. Lataa GitHub Desktopin asennusohjelma ja aloita asennus
  2. Asennuksen yhteydessä sinua pyydetään kirjautumaan GitHub-tilillesi
    • klikkaa Sign in to GitHub.com nappia
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjautunut sisään, voit siirtyä suoraan kohtaan 4. Käyttäjän valitseminen
  3. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti ja salasana
    • Klikkaa Sign in
    • Anna kaksivaiheisen tunnistautumisen avainkoodi tunnistautumissovelluksestasi ja klikkaa Verify
  4. Valitse käytettävä tili klikkaamalla Continue käyttäjänimen vierestä
  5. Anna GitHub Desktopille lupa päästä GitHub-tilillesi klikkaamalla Authorize Desktop nappia
  6. Siirry takaisin GitHub Desktopiin klikkaamalla Avaa GitHubDesktop nappia
  7. GitHub Desktop ehdottaa Gitin konfigurointia käyttäjänimen ja sähköpostin osalta. Tämä muokkaa osiossa Git:n konfigurointi käsiteltyä .gitconfig tiedostoa. Nämä tiedot liitetään kaikkiin Git committeihin.
    • Tarkasta ehdotetut käyttäjänimi ja sähköposti
    • Klikkaa Finish nappia
  8. Nyt GitHub Desktop on yhdistetty GitHub tiliisi ja käyttövalmis
    • Etusivulla näkyvät mm. GitHub tilisi repositoriot

Tässä osiossa luodaan uusi kansio ja tiedosto komentorivi-osion ohjeiden mukaisesti. Sen jälkeen kloonataan sama git-opas etärepositorio , jota on käytetty myös aiemmissa osioissa. Tämän jälkeen luodaan osiolle oma haara ja lopuksi muutokset viedään etärepositorioon. Tiedostojen muokkaukset tehdään Git Bashilla, jotta samalla pääsee harjoittelemaan komentorivin peruskomentoja.

Tiedoston luonti ja repositorioiden yhdistäminen GitHub Desktopilla

  1. Avaa Git Bash tai terminaali (macOS tai Linux)
    • Git Bash asennetaan Windowsille oletuksena Gitin asennuksen yhteydessä
    • Avaa Git Bash Windowsin aloitusvalikosta tai etsi Git Bash
  2. Luo uusi kansio testausta varten
    • Siirry hakemistoon, johon haluat luoda uuden kansion:
      cd polku/valittuun/hakemistoon
    • Luo uusi kansio projektia varten
      mkdir kansion-nimi
    • Voit ketjuttaa komentoja käyttämällä niiden välissä &&
      esim. cd Desktop/ && mkdir git-testaus-github-desktop
  3. Kloonaa etärepositorio luotuun kansioon
    • Klikkaa GitHub Desktopista Clone a repository from the internet nappia
    • Valitse testi repositoriosi
    • Anna local path kohtaan polku luomaasi kansioon
    • Klikkaa Clone nappia
  4. Tarkasta GitHub Desktopista kloonaamasi repositorion tiedot
    • Yläpalkista näet nykyisen repositorion ja haaran
  5. Lisää tiedosto ja tee muutos olemassa olevaan tiedostoon Git Bashillä
    • Siirry luotuun kansioon
      cd kansion-nimi
    • Huomaa, että Git Bash tunnistaa, että kansiossa on repositorio ja näet että olet main-haarassa
    • Komennolla dir näet että kansioon on ilmestynyt etärepositorion main -haaraan aikaisemmin lisäämäsi tiedostot
    • Muokkaa olemassa olevaa README.md tiedostoa komennolla
      echo "Lisättävä muutos tähän" >> README.md
    • Lisää uusi tiedosto komennolla
      echo "Teksti sisältö tähän" >> tiedoston-nimi-tähän.md
    • Jos teit saman virheen kuin minä, niin muutetaan uuden tiedoston nimi komennolla
      mv vanha-nimi.md uusi-nimi.md
  6. Tarkista muutokset GitHub Desktopissa
    • Vasemmalla sivupalkissa on lista muutetuista tiedostoista
    • Klikkaamalla tiedostoa pääruudussa voi tarkastella kyseisen tiedoston muutoksia
  7. Luo uusi Git-haara
    • Avaa alasvetovalikko klikkaamalla nykyisen haaran nimeä
    • Klikkaa New branch nappia
    • Anna uudelle haaralle nimi nimi
    • klikkaa Create branch nappia
    • Valitse, että nykyiset muutokset tuodaan luotavaan haaraan
    • klikkaa Switch branch nappia
  8. Tiedostojen lisääminen ja commitointi
    • GitHub Desktopissa git add tiedosto.md komentoa vastaa checkboxin klikkaaminen tiedoston nimen vierestä
    • Lisää commit-viesti Summary tekstikenttään
    • Klikkaa Commit 2 files to... nappia
  9. Vie muutokset etärepositorioon
    • Klikkaa Publish branch nappia ylävalikosta tai pääruudusta
  10. Tarkista uusi haara ja tuodut muutokset GitHubissa
    • Siirry selaimella GitHubiin ja avaa repositorio
    • Klikkaa haaran valintaa jossa lukee main
    • Valitse luomasi uusi haara
    • Tarkastele uuden haaran sisältöä
