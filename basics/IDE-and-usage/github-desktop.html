Gitin käyttö GitHub Desktopilla

Tällä sivulla käsitellään Gitin käyttöä GitHub Desktopin avulla. GitHub Desktop ei ole koodieditori, joten sen kautta ei voi muokata tiedostoja. Työkalu on tarkoitettu Git-toimintoihin, kuten tiedosotjen lisääminen, vieminen ja tuominenm repositorion haaroittamiseen (branching) ja yhdistämiseen (merging). Koodieditorien käyttöä käsitellään erikseen aiemmilla sivuilla. Haitari-elementistä voit katsoa tarvittaessa ohjeet GitHub Desktopin käyttöönottoon.

GitHub Desktopin käyttöönottamiseksi käy läpi, 1.4 käyttöönotto osiossa tehtyjen toimien lisäksi, seuraavat vaiheet:

  1. Lataa GitHub Desktopin asennusohjelma ja aloita asennus.
  2. Asennuksen yhteydessä sinua pyydetään kirjautumaan GitHub-tilillesi.
    • klikkaa Sing in to GitHub.com nappia.
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjatunut sisään, voit siirtyä suoraan kohtaan: 4. Käyttäjän valitseminen.
  3. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti ja salasana
    • Klikkaa Sign in
    • Mikäli olet ottanut käyttöön kaksivaiheisen tunnistautumisen tilillesi, pyttää GitHub sinua syöttämään tunnistautumisen avainkoodin.
    • Anna avainkoodi tunnistautumissovelluksestasi ja klikkaa Verify
  4. Valitse käytettävä tili klikkaamalla Continue käyttäjänimen vierestä
  5. Anna GitHub Desktopille lupa päästä GitHub-tilillesi klikkaamalla Authorize Desktop nappia
  6. Siirry takaisin GitHub Desktopiin klikkaamalla Avaa GitHubDesktop nappia
  7. GitHub Desktop ehdottaa, Gitin konfigurointia käyttäjänimen ja sähköpostin osalta. Tämä muokkaa osiossa Git:n konfigurointi käsiteltyä .gitconfig tiedostoa. Nämä tiedot liitetään kaikkiin Git committeihin.
    • Tarkasta ehdotetut käyttäjänimi ja sähköposti
    • Klikkaa Finish nappia.
  8. Nyt GitHub Desktop on yhdistetty GitHub tiliisi ja käyttövalmis.
    • Etusivulla näkyvät mm. GitHub tilisi repositoriot.

Tässä osiossa luodaan komentorivi osion ohjeiden malliin uusi kansio, tiedosto. Kloonataan etärepositorio, jota käytettiin edellisissä osiossa paikalliseksi. Sen jälkeen tälle osiolle luodaan oma haara (branch) ja lopuksi muutokset viedään etärepositorioon. Tiedostojen muutokset tehdään tässä ohjeessa Git Bashilla, jotta samalla tulee harjoiteltua komentorivin peruskomentoja.

  1. Avaa Git Bash tai terminaali (macOS tai Linux).
    • Git Bash asennetaan Windowsille oletuksena Gitin asennuksen yhteydessä.
    • Avaa Git Bash Windowsin aloitusvalikosta tai etsi Git Bash.
  2. Luo uusi kansio testausta varten.
    • Siirry kansioon minne haluat luoda uuden kansion:
      cd polku/valittuun/hakemistoon
    • Luo uusi kansio projektia varten:
      mkdir kansion-nimi
    • Voit ketjuttaa komentoja käyttämällä niiden välissä &&
      esim. cd Desktop/ && mkdir git-testaus-github-desktop
  3. Kloonataan etärepositorioon luotuun kansioon.
    • Klikkaa Clone a repository from the internet nappia.
    • Valitse testi repositoriosi.
    • Anna local path kohtaan polku luomaasi kansioon.
    • Klikkaa Clone nappia.
  4. GitHub Desktopissa on nyt valittuna kloonaamasi repositorio.
    • Yläpalkista näet nykyisen repositorion ja haaran.
  5. Lisätään tiedosto ja tehdään muutos olemassa olevaan tiedostoon Git Bashillä.
    • Siirry luotuun kansioon:
      cd kansion-nimi
    • Huomataan, että Git Bash tunnistaa, että kansiossa on repositorio ja näet että olet main-haarassa.
    • Komennolla dir nähdään että kansioon on ilmestynyt etärepositorion main.haaraan aikaisemmin lisätyt tiedostot.
    • Muokkaa olemassa olevaa README.md tiedostoa komennolla:
      echo "Lisättävä muutos tähän" >> README.md
    • Lisää uusi tiedosto komennolla:
      echo "Teksti sisältö tähän" >> tiedoston-nimi-tähän.md
    • Jos teit saman virheen kuin minä, niin muutetaan uuden tiedoston nimi komennolla:
      mv vanha-nimi.md uusi-nimi.md
  6. Nyt muutokset näkyvät GitHub dektopissa.
    • Vasemmalla sivupalkissa on lista muutetuista tiedostoista.
    • Klikkaamalla tiedostoa pää ruudussa voi tarkastella kyseisen tiedoston muutoksia.
  7. Uuden Git-haaran luominen.
    • Avaa alasvetovalikko klikkaamalla nykyisen haaran nimeä.
    • Klikkaa New branch nappia.
    • Anna uudelle haaralle nimi nimi.
    • klikkaa Create branch nappia.
    • Valitse, että nykyiset muutokset tuodaan luotavaan haaraan.
    • klikkaa Switch branch nappia.
  8. Tiedostojen lisääminen ja commitointi.
    • GitHub Desktopissa git add tiedosto.md komentoa vastaa valinnan tekeminen listaan tiedoston nimen viereen.
    • Lisää commit-viesti Summary tekstikenttään.
    • Klikkaa Commit 2 files to... nappia.
  9. Muutoksien vieminen etärepositorioon.
    • Klikkaa Publis branch nappia ylävalikosta tai pääruudusta.
  10. Uusi haara ja tuodut muutokset GitHubissa
    • Mene selaimella GitHubiin ja avaa repositorio
    • Klikkaa haaran valintaa jossa lukee Main
    • Valitse uusi haara
    • Aukeaa näkymä luodun haaran sisällöstä.
