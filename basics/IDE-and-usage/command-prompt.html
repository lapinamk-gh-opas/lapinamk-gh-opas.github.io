Git:n käyttö komentorivillä

Tällä sivulla käsitellään Git:n käyttöä komentoriviltä. Luodaan komentorivin avulla tiedosto ja repositorio sekä viedään se etärepositorioon.

  1. Avaa Git Bash tai terminaali (macOS tai Linux)
    • Git Bash asennetaan Windowsille oletuksena Git:n asennuksen yhteydessä
  2. Luodaan kansio ja tiedosto testausta varten.
    • Siirry kansioon johon haluat luoda uuden kansion: cd polku/valittuun/hakemistoon
    • Luo uusi kansio projektia varten: mkdir kansion-nimi
    • Siirry luotuun kansioon: mkdir kansion-nimi
    • Voit ketjuttaa komentoja käyttämällä niiden välissä &&
      esim. cd Desktop/ && mkdir git-testaus && cd git-testaus
    • Luodaan vielä markdown (.md) tekstitiedosto uuteen kansioon:
      echo Tämä on tekstitiedosto git testausta varten. >> README.md
  3. luodaan paikallinen repositorio ja yhdistetään se etärepositorioon.
    • Paikallinen repositorio luodaan kansiossa, jonka sisältö halutaan mukaan versionhallintaan komennolla: git init
    • Yhdistetään etärepositorio paikalliseen komennolla: git remote add origin https://github.com/käyttäjänimi/repositorion-nimi.git
    • Lisätään tiedosto README.md repositorioon komennolla: git add README.md
    • Tehdään ensimmäinen commit komennolla: git commit -m "Ensimmäinen commit"
  4. Lopuksi siirretään paikallisen repositorion sisältö/muutokset etärepositorioon githubissa.
    • Commitoidut muutokset saadaan siirrettyä etärepositorioon komennolla: git push -u origin main
  5. Nyt lisätty tiedosto näkyy githubissa.
    • Koska kyseessä oli README.md tiedosto myös sen sisältö näkyy repositorion etusivulla.
