Gitin käyttö Android Studiolla

Tällä sivulla käsitellään Gitin käyttöä Android Studiolla. Android Studion perustuu JetBrainsin intelliJ koodieditoriin, joten sillä on yhteneväisyyksiä myös edellä esiteltyyn JetBrainsin PyCharm editoriin. Haitari elementistä voit katsoa tarvittaessa ohjeet Gitin ja GitHubin käyttöönottoon Android Studiossa.

Gitin ja GitHubin käyttöönottamiseksi Android Studiossa käy läpi, 1.4 käyttöönotto osiossa tehtyjen toimien lisäksi, seuraavat asetukset.

  1. Klikkaa vasemmalta alhaalta Ratas kuvaketta ja valitse Settings
  2. Valitse asetusvalikon vasemmalta puolelta listasta Version Control ja sen alta Git
    • Testaa, että Android Studio löytää Gitin painamalla Test nappia.
    • Mikäli Git löytyy näykyy sen version numero polun alla.
    • Mikäli Gitiä ei löytynyt anna Android Studiolle polku johon olet Gitin asentanut.
  3. Valitse asetusvalikon vasemmalta puolelta listasta Version Controlin alta GitHub
    • Yhdistä GitHub tilisi Android Studioon klikkaamalla Add account... tekstiä.
    • Valitse Log in via GitHub...

    GitHub-tilin yhdistäminen editoriin voi hieman erota tässä esitetystä, mikäli olet jo antanut oikeudet jollekin toiselle JetBrainsin editorille käyttää tilisi sisältöä.

  4. Avautuu selain ikkuna ja klikkaa Authorize in GitHub nappia
    • Mikäli sinulla on jo GitHub auki selaimessa ja olet kirjatunut sisään, voit siirtyä suoraan kohtaan: 7. Luvananto Android Studiolle.
  5. Syötä aukeavaan selainikkunaan GitHub-tilin käyttäjänimi tai sähköposti ja salasana
    Klikkaa Sign in
  6. Anna kaksivaiheisen tunnistautumisen avainkoodi tunnistautumissovelluksestasi
    Klikkaa Verify
  7. Anna Android Studiolle lupa päästä GitHub-tilillesi klikkaamalla Authorize JetBrains

    Mikäli sinulla oli GitHub avoinna selaimessa ja olit jo valmiiksi kirjautuneena sinne sisään ja jos sinulla on kaksivaiheinen tunnistautuminen käytössä saattaa GitHub tämän vaiheen jälkeen pyytää vielä syöttämään tunnistautumissovelluksen avainkoodin (kts. kohta 6).

  8. Tulee ilmoitus, että olet sallinut Android Studiolle pääsyn GitHub-tilillesi ja voit sulkea selainikkunan.
  9. Myös Android Studio näyttää GitHub asetusvälilehden alla lisäämäsi tilin.
    Sulje asetusikkuna klikkaamalla OK nappia.

Muutoin osion ohjeissa käytetään jo komentokehote ja VS Code käytöstä tuttua etärepositoriota, Kloonataan se paikalliseksi repositorioksi IDE:llä. Luodaan editorin avulla jälleen tiedostot ja viedään ne etärepositorioon omaan haaraansa.

Huomiona mainittakoon, että myös Android Studio tarjoaa mahdollisuuksia tehdä Git toimintoja usealla eri tavalla. Jotkin toiminnoista voi tehdä mahdollisesti myös toisella tapaa, ehkäpä jopa paremmin sekä tehokkaammin kuin mitä ohjeessa on esitetty.

  1. Kloonaa etärepositorio PyCharmiin
    • Klikkaa projects valikossa More Actions .
    • Tai mikäli sinulla on aikaisempia projekteja niin niiden yläpuolelta oikeanpuolimmaisesta kolmepistettä napista
    • Klikkaa aukeavasta alasvetovalikosta Get from Version Control..
  2. Anna kloonattava repositorio.
    • Repository URL valikossa anna URL-kenttään etärepositorion osoite.
    • Muuta halutessasi kohde kansio Directory-kenttään.
    • Klikkaa Clone nappia.
    • Jos tiedät, että kansio on luotettava (esim. oma projektisi), valitse Trust Project.
    • Jos et ole varma kansion tai projektin sisällöstä ja luotettavuudesta, voit avata kansion klikkaamalla Preview in Safe Mode-nappia, jolloin voit tarkastella sisältö ennen siihen luottamista.
  3. Lisätään uudet tiedostot
    • Projekti aukeaa Android Studiossa ja näet kloonatun etärepositorion sisällön. Lisäksi Android Studio loi automaattisesti .idea asetuskansion.
    • lisää uusi tiedosto valitsemalla -ikonilla aukeavasta alasvetovalikosta File.
    • Lisättään projektiin .gitignore tiedosto antamalla lisättävälle tiedoslle nimi .gitignore ja paina Enter.
    • Android Studio kysyy jo tässä vaiheessa haluatko lisätä tiedoston versionhallintaan. Voi lisätä sen nyt painamalla Add nappia tai klikata Cancel nappia ja lisätä sen myöhemmin esitellyllä tavalla
    • Lisättään projektiin vielä toinen tiedosto antamalla lisättävälle tiedoslle nimi esim. testitiedosto-android-studiolla.md ja paina Enter.
    • Voi taas lisätä tiedoston versionhallintaa kuten kohdassa 4 tai jättää sen myöhempään.
    • Lisää tiedostolle sopiva sisältö ja tallenna pikanäppäimellä CTRL + S
    • Muuta vielä README.md tiedoston sisältöä ja tallenna pikanäppäimellä CTRL + S
  4. Jätä .idea kansio pois versionhallinnasta
    • Lisää gitignore-tiedostoon .idea/ ja tallenna pikanäppäimellä CTRL + S
    .gitignore:sta lisää tietoa osiossa: 2.5 yksityisyys ja tietoturva
    • Tiedostojen/kansioiden värit:
      Oranssi .gitignorattu versionhallinnasta
      Sininen versionhallinnassa ja muutettu
      Punainen uusi tiedosto ei lisätty versionhallintaan.
  5. Uuden haaran luominen ja uusien tiedostojen lisääminen staging tilaan.
    • Valitse New Branch joko yläpalkin versionhallinta pikavalikosta tai päävalikosta Main Menu > Git > New Branch tai pikanäppäimellä CTRL + ALT + N
    • Anna haaralle nimi, esim. android-studio-haara ja klikkaa Create.
    • Klikkaa kansiota jossa tiedostot ovat.
    • Tämän jälkeen klikkaa ylävalikosta Main Menu kuvaketta.
    • Valitse päävilikosta välilehti Git, Sen alta Selected Directory ja klikkaa Add.
    • Tämä lisää kaikki kansion tiedostot staging tilaan.
    • Vaihtoehtoisesti voit tehdä lisäämisen pikanäppäimillä painamalla kansion valitsemisen jälkeen CTRL + ALT + A. Tai voi valita kansion sijaan yksittäisen tiedonston ja viedä sen staging tilaan, joko valikosta (Main Menu > Git > Current file > Add) tai pikanäppäimillä.
    • Tämä vastaa komentoa: git add .
    • Käytä komentoa git add . varoen.
      Jos olet lisännyt samaan kansioon jotain arkaluontoistatietoa sisältäviä tiedostoja, joita et halua lisätä repositioon, niin älä käytä git add . komentoa tai lisää repositorioon .gitignore -tiedosto, joka estää siihen nimettyjen tiedostojen lisäämisen repositorioon.
    • .gitignore:sta lisää tietoa osiossa: 2.5 yksityisyys ja tietoturva
    • Lisättyjen tiedostojen väri muuttuu punaisesta vihreäksi. Muutetut säilyvät sinisenä.
  6. Muutoksin commitoiminen, viestin lisääminen ja vienti etärepositorioon.
    • Avaa Commit valikko: klikkaamalla sivupalkin Commit kuvaketta tai ylävalikosta Main Menu > Git > Commit tai pikanäppäimillä CTRL + K.
    • Valitse tiedostot committiin.
    • Lisää commit-viesti tekstikenttään.
    • Paina Commit nappia commitoidaksesi muutokset paikalliseen repositorioon.
    • tai paina Commit and Push nappia commitoidaksesi muutokset paikallisesti ja viedäksesi muutoset myös etärepositorioon.
    • Commitoitoitujen tiedostojen väri muuttuu vihreäksi valkoiseksi. Versionhallinnan ulkopuolelle jätetyt tidostot säilyvät oransseina.
    • Mikäli teit vain commitin, voit viedä commitoidut muutokset etärepositorioon tekemällä pushin erikseen
    • Valitse Push joko yläpalkin versionhallinta pikavalikosta tai päävalikosta Main Menu > Git > Push tai pikanäppäimellä CTRL + SHIFT + K
    • Seuraava push ruutu avautuun niin Commit and Push napilla kuin erikseen pushatessa. Ruudussa näkyy pushattavan commitin viesti sekä pushattava haara sekä kohdehaara ja tiedostot.
    • Klikkaa Push nappia viedäksesi muutokset etärepositorioon.
  7. Tuodut muutokset GitHubissa
    • Mene selaimella GitHubiin ja Android Studioon yhdistämääsi repositorioon.
    • Klikkaa haaran valintaa jossa lukee Main.
    • Valitse uusi haara.
    • Nyt lisätyt tiedostot näkyvät uudessa haarassa.
