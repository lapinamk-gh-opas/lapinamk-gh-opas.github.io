Gitin konfigurointi

  1. Avaa Git Bash tai terminaali (macOS tai Linux)
    • Git Bash asennetaan Windowsille oletuksena Gitin asennuksen yhteydessä
    • Avaa Git Bash Windowsin aloitusvalikosta tai etsi Git Bash
  2. Tarkasta .gitconfig tiedoston sisältö
    • Anna komento git config --global --list

    Tämän tiedoston asetuksia käytetään laitteella globaalisti, eli kaikissa git repositorioissa. Asetuksia on mahdollista muuttaa tarvittaessa myös repositorio kohtaisesti, siirtymällä kansioon jossa repositorio on ja antamalla komennot ilman --global lippua.

    Jos et ole aikaisemmin tehnyt tiedostoon muutoksia, niin se ei sisällä vielä muita asetuksia kuin asennuksen yhteydessä asetettu oletus tekstieditori (core.editor). Tässä tapauksessa se on gitin asennuksen yhteydessä annettu Visual Studio Code. Oletus editoria käytetään commit-viestien kirjoittamiseen, mikäli et ole lisännyt viestiä commit komentoon lipulla -m "commit viestisi".

  3. Lisätään tiedostoon käyttäjänimi ja sähköposti
    • Anna sähköposti komennolla git config --global user.email "sinun@sähköposti.com"
    • Anna nimi komennolla git config --global user.name "Sinun Nimi"
    • Tarkasta muutokset komennolla git config --global --list

    Git käyttää nimeä ja sähköpostia commitin ns. tekijätietoina. Mikäli käytät GitHub tilillä olevaa sähköpostiosoitetta, osaa GitHub yhdistää commitin sinun tiliisi. Tätä tiedostoa ei kuitenkaan käytetä GitHubissa tunnistautumiseen.

  4. Asetetaan oletushaaran nimeksi main
    • Komennolla git config --global init.defaultBranch main
    • Tarkasta muutokset komennolla git config --global --list

    Tämä tehtiin jo asennuksen yhteydessä system tasolle, mutta nimeäminen voidaan varmistaa vielä käyttäjätasolla tai nimeä voi muuttaa tarvittaessa myöhemmin näin.

Näillä asetuksilla pääsee hyvin alkuun. Lisää konfiguraatiosta voi katsoa git:n dokumentaatiosta komennolla:
git help config.

Dokumentaatio voi vaikuttaa alkuun aavistuksen kryptiseltä. Joten tässä vielä taulukko jo käytetyistä ja muutamista muista mahdollisesti hyödyllisistä asetuksista:

Asetus Kuvaus
user.name Commitin tekijän nimi
user.email Commitin tekijän sähköposti
core.editor Oletus editorin asettaminen commit-viesteille
init.defaultBranch Pää/oletushaaran nimeäminen
core.autocrlf Rivinvaihtojen käsittely Windows/Linux
color.ui Värit Git-komentojen tulosteessa
alias.* Lyhenteiden (alias) luominen usein käytetyille komennoille
(esim. git config --global alias.ci commit luo komenolle commit lyhenteen ci)
